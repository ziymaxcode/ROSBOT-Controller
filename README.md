Android ROS HMI Controller 🤖📱

A modern, robust Android Human-Machine Interface (HMI) built with Kotlin and Jetpack Compose. This application serves as the primary control tablet for a ROS-based robotic system (Raspberry Pi + PSoC microcontroller), offering both manual override and Excel-driven automated trajectory execution.

✨ Key Features

🔌 Industrial-Grade Networking

mDNS Auto-Discovery: Automatically scans the local network to find the robot's IP address without manual entry.

Network Binding: Forces the Android OS to route traffic through the robot's local Wi-Fi hotspot, preventing the system from falling back to cellular data when there is no internet.

WebSocket ROS Bridge: Maintains a persistent, low-latency duplex connection to the Raspberry Pi via rosbridge_server (Port 9090).

App-Level Authentication: Secure login gateway preventing unauthorized WebSocket connection attempts.

🕹️ Manual Control & E-Stop

Real-Time Telemetry: Sends Target Radius and Angle while simultaneously receiving actual movement feedback from the PSoC encoders.

Live Error Calculation: Dynamically calculates and displays Radius/Angle error percentages in real-time.

Hardware E-Stop: Dedicated "Abort" functionality instantly publishes a (0, 0) vector to halt all motor activity.

📊 Excel-Driven Automation

Trajectory Parsing: Imports .xlsx files containing sequential movement commands (Radius, Angle, Delay).

Live Data Table: Renders the execution steps in a responsive Compose UI table, updating with live feedback and error margins as the robot moves.

CSV Reporting: Exports the final test results (Target vs. Actual + Error %) to a .csv file for Quality Assurance and analysis.

🛠️ Advanced System Diagnostics

Raw Firmware Intercept: Bypasses standard JSON parsing to catch raw serial text strings (e.g., # ERROR: I2C failed) generated by the PSoC hardware.

Smart Categorization: Automatically flags critical hardware failures with UI Snackbars while logging background INIT/WARN messages into a scrollable System Diagnostics console.

OS-Level Shutdown: Dedicated UI workflow to safely publish a Linux shutdown command to the Raspberry Pi, preventing SD card corruption.

🛠️ Tech Stack & Architecture

Language: Kotlin

UI Framework: Jetpack Compose (Material Design 3)

Concurrency: Kotlin Coroutines & Flow (StateFlow)

Networking: OkHttp (WebSockets), Android ConnectivityManager, NsdManager

Data Parsing: Apache POI (Excel .xlsx), kotlinx.serialization (JSON)

Robot Middleware: ROS 1 / ROS 2 (rosbridge_suite)

📡 ROS Topic Reference (For Hardware Integration)

The app expects the robot to be running rosbridge_server and interacts with the following topics:

Topic Name

Action

Message Type

Description

/manual_command

Publish

JSON

Sends {radius: Float, angle: Float}

/automation_command

Publish

JSON

Sends array of steps: slNo, radius, angle, delaySeconds

/sys_command

Publish

std_msgs/String

Sends "SHUTDOWN" to safely halt the Pi.

/telemetry

Subscribe

JSON / Raw String

Receives movement feedback OR raw PSoC # ERROR logs.

🚀 Setup & Installation

Clone the repository:

git clone [https://github.com/YourUsername/Android-ROS-HMI.git](https://github.com/YourUsername/Android-ROS-HMI.git)


Open in Android Studio: Open the cloned directory in Android Studio (Giraffe or newer recommended).

Build & Sync: Allow Gradle to sync and download dependencies (Compose, OkHttp, Apache POI).

Deploy: Connect your Android tablet or phone via USB/Wi-Fi debugging and click Run.

Hardware Requirements for Testing

To fully test this app, connect your Android device to a Wi-Fi network hosted by a Raspberry Pi running ROS and the rosbridge_websocket node on port 9090.
